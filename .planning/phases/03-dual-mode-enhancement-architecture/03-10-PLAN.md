  <action>
    Modify the `EnhancementQueue` class to fix the race condition in queue size reporting:

    1. In `EnhancementQueue.get_status()` method (line ~126), replace:
       ```python
       status = {
           'size': self.queue.qsize(),
           'max_size': self.queue.maxsize,
           ...
       }
       ```
       With:
       ```python
       # Use queue.copy() to get a snapshot of current queue contents
       # This is thread-safe and provides consistent read
       status = {
           'size': len(self.queue.copy()),  # Thread-safe snapshot
           'max_size': self.queue.maxsize,
           ...
       }
       ```

    2. For consistency, also update `EnhancementQueue.enqueue()` and `EnhancementQueue.dequeue()` to log the correct queue size:
       - In `enqueue()` (line ~107), replace `'queue size: {self.queue.qsize()}'` with `'queue size: {len(self.queue.copy())}'`
       - In `dequeue()` (line ~120), replace `'queue size: {self.queue.qsize()}'` with `'queue size: {len(self.queue.copy())}'`

    The fix uses `queue.copy()` which creates a thread-safe snapshot by:
    - Locking the queue during copy operation
    - Returning a list copy that won't change during iteration
    - Avoiding the non-thread-safe `qsize()` method that can return inconsistent values
