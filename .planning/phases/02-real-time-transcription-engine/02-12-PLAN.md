# Context from UAT Diagnosis
- UAT test 6 reports: "Text does not appear on new line. It continues to append to old text even after a long pause"
- Root cause: Line 412 in accumulating_processor.py uses `self._new_phrase_started` instead of local `phrase_start` variable
- Line 343 incorrectly resets: `self._new_phrase_started = False` (before it's used)
- Variable is reset BEFORE being captured, so phrase_start always evaluates to False
- This causes all phrases to be treated as continuation instead of new phrase
</context>
