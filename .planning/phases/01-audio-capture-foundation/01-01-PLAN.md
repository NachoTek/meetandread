  <action>
- Implement a minimal, deterministic on-disk format:
  - During recording, write raw PCM int16 mono frames to `*.pcm.part` (append-only).
  - Write a JSON sidecar `*.pcm.part.json` containing at least: sample_rate=16000, channels=1, sample_width_bytes=2.
- Provide an API suitable for capture threads:
  - `PcmPartWriter.write_frames_i16(frames: bytes) -> None` (frames already int16 little-endian mono).
  - `PcmPartWriter.flush()` and `close()`.
- Implement `wav_finalize.finalize_part_to_wav(part_path: Path, wav_path: Path) -> Path`:
  - Read the sidecar metadata.
  - Create a standard PCM WAV (44-byte header) via stdlib `wave` and copy frames.
- Implement `recovery.find_part_files(recordings_dir) -> list[Path]` and `recovery.recover_part_files(...)`:
  - Convert each `*.pcm.part` to `*.wav.recovered` (or to the final `.wav` name if safe).
  - Preserve originals by default (move to `*.pcm.part.recovered.bak`) unless `delete_original=True`.
- Do not hand-roll WAV headers unless strictly necessary; use stdlib `wave` to keep risk low.
  </action>
  <verify>python -m pytest -q</verify>
  <done>
- Writer appends frames without loading entire audio into memory.
- Finalizer creates a WAV that `wave.open(...).getnframes() > 0`.
- Recovery can produce a WAV from a leftover `.pcm.part`.
  </done>
</task>
