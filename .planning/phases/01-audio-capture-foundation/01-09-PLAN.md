  <action>
Implement a session-side frame cap so the session can enforce an audio-duration limit independent of wall-clock time.

1) Extend SessionConfig:
- Add `max_frames: Optional[int] = None` to `SessionConfig` (frames at target sample_rate, per-channel frame count).
- Update the SessionConfig docstring to explain that max_frames truncates recording output and is intended for deterministic bounded recordings.

2) Enforce cap during disk writes:
- In `AudioSession._consumer_loop()`, before writing, check `self._config.max_frames`.
 - Treat `max_frames` as a hard cap on frames written to disk. Once the cap is reached, the consumer must keep consuming frames from its input queue, but DISCARD them (do not write) so shutdown cannot deadlock.
 - Maintain a local `discard_mode: bool = False` in the consumer.
 - If `max_frames` is set and `self._stats.frames_recorded >= max_frames`:
   - Set `discard_mode=True`.
   - Continue consuming frames, but skip any writer calls.
   - Do NOT set `self._stop_event` here (stop is controlled by `AudioSession.stop()`); the goal is to cap what gets written while allowing the pipeline to keep running until stop.
 - If the next chunk would exceed the cap, write only the remaining frames then switch to discard mode:
  - `remaining = max_frames - self._stats.frames_recorded`
  - `mixed = mixed[:remaining]` (slice on axis 0)
  - Write only the truncated chunk.
   - Immediately after the final write: switch to `discard_mode=True`.

  Apply the same cap/`discard_mode` logic to the "Drain remaining frames" section at the end of `_consumer_loop()` so stop-time draining never writes past the cap.

Critical correctness constraints:
- Do NOT early-return/`break` in a way that skips finalization/cleanup; the consumer must still run any existing writer close/finalize logic (typically via `finally:`) even after the cap is hit.
- Do NOT stop consuming early; drain/consume-but-discard after cap so producers/queues cannot block waiting for the consumer.

3) Keep behavior unchanged when max_frames is None:
- Ensure the default path records as it does today.

Avoid:
- Do NOT change FakeAudioModule pacing in this plan; the fix should be robust even with non-real-time sources.
- Do NOT change sample rates or mixing logic.
